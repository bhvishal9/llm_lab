## Observations
- Threshold of 0.85 is far too aggressive — the system abstains on almost everything, including queries it should clearly answer
- Coverage drops to 0.143, meaning 85% of retrieval queries return nothing at all
- Out-of-scope abstention is perfect (1.000) but only because the threshold filters out virtually all results indiscriminately
- Multi-hop recall collapses to 0.000 — requiring all expected docs to be above 0.85 similarity is too strict

## Comparison with baseline (threshold 0.70)
| Metric | 0.70 | 0.85 | Change |
|---|---|---|---|
| Factual recall | 0.933 | 0.133 | -0.800 |
| Multi-hop recall | 1.000 | 0.000 | -1.000 |
| Abstention rate | 0.000 | 1.000 | +1.000 |
| Coverage | 1.000 | 0.143 | -0.857 |

## Conclusion
The threshold is the primary lever controlling the recall/abstention trade-off. At 0.70 the system retrieves well but never abstains. At 0.85 it abstains almost universally. The right value likely sits between these — next experiment should try 0.75 or 0.80 to find a threshold that maintains high retrieval recall while improving abstention on genuinely out-of-scope queries.
